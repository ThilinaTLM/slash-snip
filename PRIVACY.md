# Privacy Policy for SlashSnip

**Last Updated:** January 2025

## Overview

SlashSnip is a text expansion browser extension that helps you save time by expanding short triggers into longer text snippets. This privacy policy explains how SlashSnip handles your data.

## Data Collection

**SlashSnip does not collect, transmit, or share any user data.**

All your templates, settings, and usage data are stored locally on your device using Chrome's built-in storage API. No data ever leaves your browser.

## Data Storage

SlashSnip stores the following data locally on your device:

- **Templates:** Your text expansion templates, including triggers, content, and metadata
- **Groups:** Organization groups for your templates
- **Settings:** Your preferences such as trigger key configuration and UI settings

This data is stored using Chrome's `chrome.storage.local` API and remains entirely on your device.

## Permissions

SlashSnip requires the following browser permissions:

| Permission | Purpose |
|------------|---------|
| `storage` | Store your templates and settings locally on your device |
| `clipboardWrite` | Write expanded text to clipboard when using clipboard-related features |
| `activeTab` | Access the current tab to detect triggers and expand text in input fields |

## Third-Party Services

SlashSnip does not use any third-party services, analytics, or tracking tools.

## Data Sharing

SlashSnip does not share any data with third parties. All data remains local to your device.

## Data Export/Import

SlashSnip provides export and import functionality that allows you to:
- Export your templates and settings to a local JSON file
- Import templates from a JSON file

These operations are performed entirely locally and do not involve any external servers.

## Updates to This Policy

If this privacy policy is updated, the changes will be reflected in this document with an updated date.

## Contact

If you have questions about this privacy policy, please open an issue on the project's GitHub repository.
